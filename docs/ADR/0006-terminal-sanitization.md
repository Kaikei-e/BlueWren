# ADR-0006: 端末出力サニタイズ規約

- **ステータス**: 採択 (Accepted)
- **日付**: 2026-05-06
- **関連**: 設計指針 §7 / `docs/principles/THREAT_MODEL.md` A6 (悪意ある peer)

## コンテキスト

CLI チャットでは、悪意ある peer が送信する制御文字 / ANSI エスケープ
シーケンスを表示するとプロンプト偽装・画面消去・カーソル制御・
クリップボード書き換え (OSC 52) ・端末タイトル変更 (OSC 0/2) などが可能になり、
利用者を直接攻撃できる。

`docs/principles/THREAT_MODEL.md` の A6 (悪意ある peer) は、相手プロセスが
悪意を持つ場合を前提とする。受信メッセージを表示する経路は、平文を信頼しない
前提でサニタイズする必要がある。

サニタイズの実装方針には、外部クレート (`strip-ansi-escapes` 等) を使う案と、
自前パーサを書く案がある。BlueWren の規模 (数十行のパーサ) と監査性を優先する
判断は設計指針 §7.4 で示されているが、本 ADR で正式に固定する。

## 決定

### 許可される制御文字 (設計指針 §7.1)

受信メッセージを表示する前のサニタイズで許可する制御文字は **`\n` (LF, 0x0A) と
`\t` (HT, 0x09) のみ** とする。

- **`\r` (CR, 0x0D) は禁止**: 行頭復帰を起こし、画面上の偽装 (UI spoofing) に
  使えるため、削除または視覚化 (`\r` という文字列に置換) する。
- それ以外の C0 制御文字 (0x00..=0x1F, 0x7F) と C1 制御文字 (0x80..=0x9F) は
  すべて U+FFFD (置換文字) に変換する。

### マルチライン受信表示の各行プレフィクス (設計指針 §7.2)

受信メッセージが複数行にまたがる場合、最初の行だけでなく **2 行目以降にも
`[相手]` のプレフィクス** を付ける。これにより、相手が「`\n > sudo rm -rf ...`」
のようにシェルプロンプトを偽装する経路を遮断する。プレフィクスは利用者が
「これは相手から来たデータである」と視覚的に区別できる手がかりとして機能する。

### エスケープシーケンスの全消去 (設計指針 §7.3)

ESC (`0x1B`) で始まるシーケンスは、以下すべてを消費して破棄する:

- **CSI** (`ESC [ ... <0x40-0x7E>`): 画面クリア・カーソル制御に悪用される
- **OSC** (`ESC ] ... <BEL or ESC \>`): クリップボード書き換え (OSC 52)・
  端末タイトル変更 (OSC 0/2) に悪用される
- **その他の単発エスケープ** (`ESC <その他>`)

完璧な ANSI パーサは不要だが、既知の悪用パターンに対する実用的な防御を
提供する。

### 自前パーサと外部クレートの選択 (設計指針 §7.4)

BlueWren の規模 (50 行程度のパーサ) と監査性を優先し、**自前実装を採用**
する。`strip-ansi-escapes` のような外部クレートを使うと依存関係が増え、
その crate 自体の脆弱性に左右される。50 行程度の単純なパーサであれば
コードレビューで挙動を完全に把握でき、テストで主要な悪用パターンに対する
防御を直接検証できる。

## 帰結

利点:

- 悪意ある peer が直接的に利用者の端末を制御できない
- 自前パーサにより監査範囲が小さく、レビューしやすい
- 依存関係を増やさない (サプライチェーン表面積維持)
- `[相手]` 各行プレフィクスにより、シェルプロンプト偽装による誘導攻撃を
  視覚的に遮断する

代償:

- 完璧な ANSI / Unicode 対応ではない (利用者が送る合法な制御文字も U+FFFD に
  なる場合がある)
- 自前パーサのため、新しい悪用パターン (CSI / OSC の新規動作) には実装側で
  追随する必要がある
- 表示は控えめになり、リッチなフォーマットは送れない (BlueWren のスコープ
  上、これは許容)

## 将来の検討事項

- Unicode のホモグリフ攻撃 (混乱を招く文字混在) への対応は v0.3 以降で検討
- 端末ごとに異なる制御シーケンス (Kitty graphics protocol 等) への対応は
  追加 ADR を起票する
- 自前パーサに対するファジングテストの追加
