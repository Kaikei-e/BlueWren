# ADR-0001: プロジェクトスコープと非目標

- **ステータス**: 採択 (Accepted)
- **日付**: 2026-05-06 (初版), 2026-05-06 (v0.2 更新), 2026-05-06 (v0.2 audit-sync)
- **意思決定者**: BlueWren メンテナ
- **関連**: ADR-0008 (サプライチェーン hardening), ADR-0009 (timeout / stream 制限), `docs/principles/BlueWren_v0_2_design_guidelines.md`

## コンテキスト

P2P 暗号化チャット領域には Briar、SimpleX、Session、Tox など多くの先行実装が
存在する。これらの大部分はグループチャット、ファイル送信、複数識別子の管理、
連絡先の発見など、利便性のための機能を抱え、結果としてコードベースが大規模化
している。脅威モデルが暗黙のうちに拡張され、セキュリティレビューと整合性の
維持が困難になりやすい。

BlueWren は学習・実験プロジェクトとして、上記の轍を踏まずに「最小の表面積で
明確な脅威モデルを満たす」ことを優先する。

## 決定

v0.2 のスコープを以下に限定する。

**含むもの**:

- 1対1 のテキストチャット
- 揮発的アイデンティティ (プロセスごとの Ed25519 鍵)
- 帯域外チケット交換 (stdin プロンプトをデフォルト、`--unsafe-ticket-arg` のみ互換用に保持)
- 256 バイト固定長アプリケーションフレーム (version + type 付き、予約範囲 `0x80..=0xFF` を拒否)
- SAS / pairing code (≥ 24 bit、推奨 30 bit) による相手認証。TLS exporter にバインドし、番号再入力で確認 (ADR-0004 参照)
- 端末出力 sanitize による escape injection 対策 (ADR-0006 参照)
- 全 await 点に timeout 必須 (起動 10〜15s、対話 5〜10min、session max 1h) と
  ストリーム数 1 双方向 / 0 単方向の構造的制限 (ADR-0009 参照)
- ターミナル UI (CLI)
- サプライチェーン hardening: `Cargo.lock` のコミット必須、`cargo audit` / `cargo deny check` の CI 必須化、
  `cargo auditable build --release --locked` でのリリースビルド、`quinn-proto >= 0.11.14`
  (RUSTSEC-2026-0037 / CVE-2026-31812) 維持の確認 (ADR-0008 参照)
- ログ・メタデータ境界: デフォルト `bluewren=warn,iroh=warn`、
  メタデータ出力は `--unsafe-debug-log-metadata` 必須 (ADR-0007 参照)

**含まないもの (非目標)**:

- 多対多のグループチャット
- ファイル・画像・音声の送信
- メッセージ履歴の永続化
- 長期アイデンティティ
- GUI クライアント
- モバイル対応
- IP 匿名化 (Tor / I2P 統合)
- 通信タイミング隠蔽 (cover traffic)
- グローバルパッシブ攻撃者への耐性
- 完全なメモリフォレンジック耐性
- 端末・OS・clipboard・swap までを含む完全な非永続化

## 帰結

利点:

- コードベースが小さく、レビューと脅威モデル検証が容易
- 機能ごとに独立したセキュリティ責務を負わずに済む
- 「何を守って何を守らないか」の説明が明確
- 「匿名性ツール」と誤認させない誠実な OSS 表現

代償:

- 多くの実用シナリオはカバーできない
- グループチャット・ファイル送信を欲するユーザーは別ツールを使う必要がある
- ジャーナリズム等の高リスク用途には Tor 統合や別カテゴリツールが必要

## 将来の検討事項

v0.3 以降で検討する候補（優先度順）:

1. 独自リレー運用と `--relay-mode` の実装 (ADR-0005 参照)
2. Tor (`arti-client`) 統合による IP 匿名化 (Iroh モードと同じ「匿名性」ラベルで混合せず、`bluewren onion ...` 等の別実行モードとして分離。ADR-0005 §「別実行モード」参照)
3. 任意のオプトイン長期鍵サポート (導入時は ADR-0002 を更新または別 ADR で上書き、脅威モデルへの影響を明示)
4. cover traffic (実験機能として、グローバル攻撃者対策ではない位置づけで)
